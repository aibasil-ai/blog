# 個人品牌網站技術選型建議書

本文件基於對標竿網站 **[Jackle.pro](https://jackle.pro)** 與 **[Haosquare.com](https://haosquare.com)** 的技術分析，並依您的指定採用 **方案 C**，規劃打造高品質部落格的技術路徑。

## 1. 標竿案例分析

| 特徵 | [Haosquare.com](https://haosquare.com) (好豪筆記) | [Jackle.pro](https://jackle.pro) (陳泰呈) |
| :--- | :--- | :--- |
| **主要定位** | 純文字部落格、知識輸出 | 個人品牌、作品集、Web 工具展示 |
| **核心技術** | **Hugo** (Go 語言靜態生成) | **Nuxt 3** (Vue.js 全端框架) |
| **樣式系統** | Tailwind CSS | Tailwind CSS |
| **互動程度** | 低 (以閱讀體驗為主) | 高 (含動態應用、工具互動) |
| **優勢** | 極致的載入速度、維護簡單 | 功能擴充性強、體驗現代化 |

## 2. 技術路徑方案比較

根據您的開發背景與需求，我們歸納出三種可行路徑：

### 方案 A：極致效能派 (類似 Haosquare)
*   **核心引擎**：**Hugo**
*   **適合對象**：專注於高頻率寫作，希望網站「快到飛起」，不需要登入系統或複雜工具。
*   **優點**：建構速度世界最快，生成的網站極小，完全靜態，安全性最高。
*   **缺點**：擴充動態功能（如串接 API 的工具）較困難，需脫離框架寫原生 JS。

### 方案 B：全能應用派 (類似 Jackle.pro)
*   **核心引擎**：**Next.js (React)** 或 **Nuxt 3 (Vue)**
*   **適合對象**：前端開發者，網站不只是部落格，未來想掛載各種 SaaS 工具或實驗性功能。
*   **優點**：生態系最豐富，可直接開發複雜互動應用（如 AI 對話、SaaS 服務）。
*   **缺點**：JavaScript Bundle 較大，若未優化，初次載入效能可能不如純靜態網站。

### 方案 C：現代平衡派 (🏆 指定採用)
*   **核心引擎**：**Next.js (React)**
*   **適合對象**：希望保有現代前端的開發體驗、效能與可擴充性，同時維持部落格的簡潔結構。
*   **優點**：
    *   **路由與內容整合**：檔案式路由與 SSG/SSR，內容頁與文章架構清晰。
    *   **元件化**：以 React 建構版型與內容區塊，易於擴充與重用。
    *   **內容友善**：可使用 Markdown/MDX 與資料檔 (JSON/YAML) 管理文章。

## 3. 建議採用的黃金技術棧

以下為方案 C 的具體落地組合（依您的指示調整）：

| 層級 | 建議技術 | 選擇理由 |
| :--- | :--- | :--- |
| **框架 (Framework)** | **Next.js (React)** | 以 Next.js 建立部落格與內容頁，路由清晰並支援靜態輸出。 |
| **語言 (Language)** | **TypeScript** | 確保程式碼品質與長久維護性。 |
| **樣式 (Styling)** | **Tailwind CSS** | 開發速度快，原子化 CSS 易於維護，也是上述兩個標竿網站的共同選擇。 |
| **UI 元件庫** | **Shadcn/ui** | 目前最流行的元件庫。它不是 npm 套件，而是將源碼複製到專案中，擁有極高的客製化自由度與設計質感。 |
| **內容管理** | **MDX** | 讓您在 Markdown 文章中直接使用 React 元件，實現「互動式文章」。 |
| **版本控管** | **Git** | 以 Git 管理版本，方便回溯、協作與部署。 |
| **套件管理** | **Yarn** | 依指定使用 Yarn，鎖定依賴版本，提升安裝穩定性。 |

## 4. Haosquare 風格指南（部落格視覺方向）

以下為參考 **Haosquare.com** 的閱讀體驗整理，作為設計與切版的共通準則：

*   **版面結構**：單欄為主，最大內容寬度約 680–760px，左右留白寬，頁面上緣保留明顯呼吸感。
*   **字體策略**：標題使用具書卷感的襯線字，內文使用清晰的無襯線字；標題與內文字級有明顯層次（例如 32/24/18/16）。
*   **閱讀節奏**：行高偏大（約 1.7–1.9），段落間距明顯，內容聚焦閱讀，不塞滿側欄與雜訊元素。
*   **色彩規劃**：以黑/深灰為主、低飽和強調色點綴，背景保持乾淨，避免大面積高彩度。
*   **連結與互動**：連結採低調但明確的視覺（底線或淡色塊），Hover 變化小而清楚。
*   **內容呈現**：標題前後留白、列表簡潔、引用與程式碼區塊使用淡底色區分，強調可讀性。

## 5. 執行與實作建議

### 第一階段：基礎建設
1.  **專案初始化**：使用 Next.js 建立 React 專案並初始化 Git。
    ```bash
    yarn create next-app blog --ts
    cd blog
    git init
    yarn
    ```
2.  **整合 Tailwind 與 Shadcn**：建立共用的設計系統（字體、顏色、基礎元件）。
3.  **內容結構**：建立 `Blog` (文章)、`Pages` (靜態頁) 與 `Assets` 的資料夾結構。
4.  **視覺風格**：採用 **Haosquare.com** 風格，偏向極簡、留白充足、聚焦閱讀體驗。

### 第二階段：功能強化
1.  **作品集展示**：設計專屬的作品展示頁面，利用 Grid 佈局呈現專案截圖與技術標籤。
2.  **SEO 優化**：設定 Sitemap、Robots.txt 與 Open Graph (社群分享圖卡)。
3.  **深色模式**：實作類似 Jackle.pro 的一鍵切換日夜間模式。
4.  **部署與發布**：後續再決定採用 **Vercel** 或 **Cloudflare Pages**，建立 CI/CD 與全球 CDN。

### 第三階段：進階整合 (視需求)
1.  **Web 工具開發**：利用 Astro 的島嶼架構，在 `/tools` 路徑下開發 React/Vue 小工具。
2.  **留言/互動系統**：整合 Giscus (GitHub Discussions) 或自建 Supabase 後端。
